================================================================================
AIaW (AI as Workspace) 总体实现报告
================================================================================

1. 项目架构总览
--------------------------------------------------------------------------------

技术栈:
- 前端框架: Vue 3 (Composition API) + TypeScript
- UI 框架: Quasar Framework
- CSS: UnoCSS (原子化 CSS)
- 状态管理: Pinia (7 个 store)
- 本地数据库: Dexie (IndexedDB 封装)，可选 Dexie Cloud 同步
- AI SDK: Vercel AI SDK (`ai` 包)，支持 streamText/generateText
- 构建: Quasar CLI (基于 Vite)
- 国际化: vue-i18n，支持 en-US / zh-CN / zh-TW
- JSON Schema 验证: @cfworker/json-schema
- 插件: 支持 LobeChat、Gradio、MCP 三种插件格式
- 模板引擎: Nunjucks (template-engine.ts)

目录结构:
src/
  pages/         — 页面组件 (10 个)
  views/         — 子视图组件 (11 个)
  components/    — 全局组件 (60+)
  stores/        — Pinia store (7 个)
  composables/   — 组合函数 (25+)
  utils/         — 工具函数 (20+)
  boot/          — Quasar 启动插件 (i18n, global-components, unocss)
  router/        — 路由定义
  layouts/       — 布局组件 (MainLayout)
src-backend/     — Python 后端 (app.py)
src-tauri/       — Tauri 桌面端壳 (Rust)
src-capacitor/   — Capacitor 移动端壳 (Android/iOS)
src-pwa/         — PWA 配置 (manifest.json, service worker)

平台适配策略:
- Web: 直接运行，使用 window.fetch
- Tauri (macOS/Windows/Linux): 使用 tauriFetch 替代 fetch，支持 MCP stdio 插件
- Capacitor (Android/iOS): 使用 capacitor-stream-fetch，iOS 回退 window.fetch
- 平台检测: IsTauri / IsCapacitor / IsWeb 常量 (src/utils/platform-api.ts)


2. 数据流全景
--------------------------------------------------------------------------------

用户输入 → AI 响应完整链路:

1. 用户在 DialogView 的 contenteditable div 输入文本
2. 按 Enter (或其他配置的发送键) 触发 send()
3. send() 校验: 助手是否设置、模型是否解析成功
4. resolveRuntimeSdkModel() 解析 SDK 模型对象
5. stream() 函数:
   a. 构建系统提示 (getSystemPrompt)，注入插件 prompt
   b. 收集启用插件的 tools，注册为 Vercel AI SDK tool()
   c. 调用 streamText() 或 generateText() (根据 assistant.stream 设置)
   d. 流式接收 text-delta / reasoning-delta，节流更新到 Dexie (50ms)
   e. 工具调用: callTool() → callApi() → 插件 execute() → 结果存入 items 表
   f. MiniMax/Cerebras 特殊后处理
6. 结果写入 db.messages，状态从 streaming → default/failed
7. 可选: autoGenTitle 生成对话标题，autoExtractArtifact 提取 artifact


3. 状态管理架构
--------------------------------------------------------------------------------

stores/workspaces.ts — 工作区/文件夹 CRUD，递归删除含子级、关联 dialog/message/item/assistant/artifact
stores/providers.ts  — 自定义 Provider 管理，subprovider 模型映射，支持 fallback 链
stores/assistants.ts — 助手 CRUD，关联 workspaceId
stores/plugins.ts    — 插件生命周期 (安装/卸载/刷新)，支持 LobeChat/Gradio/MCP 三种类型
stores/user-data.ts  — 用户运行时状态 (lastWorkspaceId, tip 关闭记录等)
stores/user-perfs.ts — 用户偏好设置 (provider/model/主题/快捷键/artifacts 等)
stores/ui-state.ts   — UI 临时状态 (mainDrawerOpen, dialogScrollTops)

关键 composable:
- persistentReactive(key, default) — 基于 Dexie reactives 表的响应式持久化
- useLiveQuery(querier) — Dexie liveQuery → Vue shallowRef 响应式绑定
- syncRef(source, setter) — 双向同步 ref：source 变化更新 val，val 变化调用 setter

5. 跨平台实现策略
--------------------------------------------------------------------------------

fetch 封装 (src/utils/platform-api.ts):
- Tauri: tauriFetch (src/utils/tauri-stream.ts) — 绕过 CORS
- Capacitor Android: capacitor-stream-fetch
- Capacitor iOS: window.fetch (stream-fetch 未实现)
- Web: window.fetch

剪贴板:
- Tauri: @tauri-apps/plugin-clipboard-manager
- Capacitor: @capacitor/clipboard
- Web: navigator.clipboard

文件导出:
- Tauri/Capacitor: 各自文件系统 API
- Web: Quasar exportFile (download)

MCP 插件: 仅 Tauri 支持 stdio transport (需桌面端原生进程)


6. 插件系统架构
--------------------------------------------------------------------------------

三种插件格式:
1. LobeChat Plugin — 兼容 LobeHub 插件生态，manifest 含 api 列表
2. Gradio Plugin — 调用 HuggingFace Gradio 接口
3. MCP Plugin — Model Context Protocol，支持 SSE 和 stdio transport

插件数据流:
- 安装: manifest 存入 installedPluginsV2 表，默认配置存入 reactives 表
- 运行: DialogView 中 activePlugins 过滤已启用的 → 构建为 AI SDK tool → stream 中调用
- 工具调用结果: 存为 StoredItem (items 表)，通过 content 引用关联到消息

内置插件: webSearch, calculator, time, whisper, flux, emotions, mermaid, docParse, artifacts, codeExec, fileOps, localFsNative

插件配置层级:
- 全局: PluginSettings (per-plugin settings + fileparser)
- 助手级: PluginAdjust (per-tool enable/disable, 参数, prompt vars)


7. 数据持久化策略
--------------------------------------------------------------------------------

Dexie 数据库 'data' (IndexedDB):
- workspaces (id, type, parentId) — 工作区+文件夹树
- dialogs (id, workspaceId) — 对话，含 msgTree (消息树) 和 msgRoute (当前路径)
- messages (id, type, dialogId) — 消息，含 contents 数组 (多种类型)
- assistants (id, workspaceId) — 助手配置
- artifacts (id, workspaceId) — Artifact (代码片段/文档)，含版本历史
- installedPluginsV2 (key, id) — 已安装插件
- reactives (key) — 响应式持久化 KV (user-data, user-perfs, plugins-data)
- avatarImages (id) — 头像图片
- items (id, type, dialogId) — 工具调用结果项
- providers (id) — 自定义 Provider

可选 Dexie Cloud: 配置 DexieDBURL 后启用，提供云同步和用户认证


8. 国际化方案
--------------------------------------------------------------------------------

- vue-i18n，初始化在 src/boot/i18n.ts
- 语言文件在 src/i18n/ 目录
- 支持: en-US, zh-CN, zh-TW
- 用户可在设置中切换，或跟随系统
- 市场 JSON 文件按语言分: /json/assistants.{locale}.json, /json/plugins.{locale}.json
