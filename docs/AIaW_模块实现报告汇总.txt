DialogView — 对话视图（核心页面）
================================================================================
文件路径: src/views/DialogView.vue (2323 行)

1. 功能概述
AI 对话的核心交互页面，处理消息发送、流式响应、工具调用、Artifact 提取、
消息树分支导航、自动标题生成等完整对话生命周期。

2. 入口与路由
- URL: /workspaces/:workspaceId/dialogs/:dialogId
- Props: id (dialogId)
- 注入: workspace (Ref<Workspace>), dialogs (Ref<Dialog[]>), artifacts (Ref<Artifact[]>)

3. 核心数据流
1. useLiveQueryWithDeps 监听 dialog 和 messages 变化
2. syncRef 将 liveDialog → dialog (双向同步，写入时通过 db.dialogs.update)
3. messageMap / itemMap 由 liveMessages / liveItems 构建的查找表
4. chain (消息链) 通过 getChain('$root', msgRoute) 递归构建
5. 用户输入 → send() → stream() → streamText/generateText → 节流更新 contents → db.messages.update
6. 工具调用: callTool() → callApi() → 插件 execute() → 结果存 items 表

4. 依赖关系
- Stores: useAssistantsStore, useUserPerfsStore, usePluginsStore, useUserDataStore, useUiStateStore
- Composables: useLiveQueryWithDeps, syncRef, useCallApi, useCreateDialog, useCreateArtifact,
  useMdPreviewProps, useSetTitle, useLocateId, useListenKey
- Components: ViewCommonHeader, UnifiedInput, AssistantItem, ModelItem, AutocompleteInput 等
- 外部: ai (Vercel AI SDK - streamText, generateText, tool), mark.js, nunjucks

5. 关键实现细节

消息树结构:
- dialog.msgTree: Record<string, string[]> 树结构，$root 为根
- dialog.msgRoute: number[] 当前选中路径
- chain: 当前显示的消息 ID 列表
- getChain(node, route) 递归遍历树，返回 [chain, normalizedRoute]

分支导航:
- switchChain(index, value) 切换分支
- regenerate(index) 在指定位置重新生成（添加分支）
- edit(index) 编辑用户消息（创建 draft 分支）

流式发送 (stream 函数, 行 1243):
1. 构建 assistant 消息，追加到 msgTree
2. 创建下一条 user 消息 (status: 'inputing') 作为下次输入
3. 收集 activePlugins，构建 AI SDK tools
4. 构建系统提示 (getSystemPrompt)，注入插件 prompt
5. 组装 messages (getChainMessages)，添加 system prompt
6. 调用 streamText 或 generateText
7. 流式接收 text-delta，throttle(50ms) 更新 Dexie
8. 工具调用结果存入 items 表
9. MiniMax 特殊处理: 流错误回退 generateText, think 标签提取, 空文本补发
10. Cerebras 特殊处理: 去除首尾空白

模型解析:
- resolveRuntimeSdkModel() → sdkModel (computed) 或 resolveCustomSdkModelFallback()
- sdkModel 从 assistant.provider/model → perfs.provider/model 逐级回退

滚动锁定:
- lockingBottom: 流式输出时自动滚到底部，用户向上滚动时解锁
- requestAnimationFrame 避免频繁滚动

目录导航:
- activeCatalogMessageId 追踪视口中的 assistant 消息
- showDesktopCatalog 在宽屏时显示消息目录

6. 与其他模块交互
- 使用 workspace/dialogs/artifacts 注入数据
- 调用 useCallApi 执行插件
- 调用 useCreateArtifact 提取 artifact
- 调用 useCreateDialog 创建新对话
- 通知 pluginsStore 获取可用插件
- 使用 perfs 中的 streamRenderLevel / artifactsAutoExtract 等配置

功能概述
本组组件负责各种设置表单、输入控件、对话框等 UI 构建。大部分是通用的表单组件，被多个视图复用。

核心组件

1. TypesInput.vue（256行）— 通用类型输入
   - 根据 JSON Schema type 渲染对应控件：string→Input/Select, number→Input(type=number), boolean→Checkbox, array→ListInput, object→JsonInput
   - 是 JsonInput 和 PluginSettings 的基础渲染引擎

2. JsonInput.vue（64行）— JSON Schema 驱动表单
   - 遍历 schema.properties，为每个属性渲染 TypesInput
   - component prop 控制外层包装（item 或 input）

3. ModelsInput.vue（33行）— 模型选择器
   - Quasar Select + useFilterOptions，支持搜索和新建自定义模型

4. ProviderInputItems.vue（159行）— 提供商选择
   - 提供商类型选择 + API Key/Base URL 配置
   - 集成 GetModelList 获取远程模型列表

5. SubproviderInput.vue（44行）— 子提供商编辑
   - 提供商选择 + modelMap 编辑（源模型→目标模型映射）

6. CustomProviders.vue（101行）— 自定义提供商列表
   - 展示所有自定义提供商，支持添加/删除/编辑

7. GetModelList.vue（108行）— 远程模型列表获取
   - 调用 providersStore.getModelList 获取提供商支持的模型

8. PickAvatarDialog.vue（261行）— 头像选择弹窗
   - 支持三种头像类型：文字、图标、图片上传
   - 图标选择器带分类和搜索；Hue 滑块调色

辅助组件
- LazyInput：延迟更新（@change 而非 @input）
- ListInput：列表编辑（Quasar Select with add-unique）
- VarsInput：键值对变量编辑
- PromptVarInput/PromptVarItem：插件提示词变量编辑
- AutocompleteInput：自动补全输入
- ShortcutKeyInput：快捷键录制输入
- PlatformEnabledInput：平台开关（always/desktop-only/never）
