模块名称：Composables 层 — 数据响应与持久化
文件路径：src/composables/

功能概述
本组 composable 负责将 IndexedDB 数据库查询转化为 Vue 响应式数据，并实现自动持久化机制。是整个应用数据层的基石。

包含的 Composable

1. live-query.ts — Dexie LiveQuery 响应式包装
   - useLiveQuery(querier, options)：将 Dexie liveQuery 订阅包装为 ShallowRef，数据库变更自动反映到 Vue 组件
   - useLiveQueryWithDeps(deps, querier, options)：支持响应式依赖，依赖变化时重新创建查询
   - 关键实现：L31-47，创建 observable → subscribe → shallowRef 赋值；getCurrentScope/onScopeDispose 自动清理
   - 被所有 store 使用

2. persistent-reactive.ts — 响应式对象自动持久化
   - persistentReactive<T>(key, value)：返回 [reactive, ready]
   - 机制：reactive watch → db.reactives.put；useLiveQuery 监听 → Object.assign 更新
   - flag 机制（L8, L11-13）防止 watch 循环：DB 写入触发 LiveQuery → 赋值 → 触发 watch → 再次写入
   - 被stores/user-perfs、user-data、plugins 使用

3. sync-ref.ts — 双向数据绑定
   - syncRef<T>(source, set, options)：创建 ref 与 source 双向同步
   - 用途：将 inject 的对象与 store 双向绑定，视图修改自动持久化
   - flag 机制防止循环更新
   - 被 WorkspaceIndex、WorkspaceSettings、AssistantView、PluginAdjust、CustomProvider 使用

4. local-reactive.ts — LocalStorage 响应式
   - localReactive<T>(key, value)：基于 Quasar LocalStorage 的响应式对象
   - 监听 storage 事件实现跨标签页同步
   - 用于不需要 IndexedDB 的轻量持久化

核心数据流
IndexedDB ←useLiveQuery→ ShallowRef ←Object.assign→ reactive(对象) ←v-model→ 视图
                                        ↑persistentReactive watch─────────────┘

与其他模块的交互
- 所有 Store：使用 useLiveQuery 获取数据
- persistent-reactive：user-perfs、user-data、plugins store 的数据持久化
- sync-ref：视图与 store 的双向绑定
