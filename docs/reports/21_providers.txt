模块名称：stores/providers.ts（提供商状态管理）
文件路径：src/stores/providers.ts

功能概述
管理自定义提供商（CustomProvider）。核心功能是提供商工厂：根据 Provider 配置创建 SDK 提供商实例（函数），支持原生提供商和自定义提供商的递归解析。自定义提供商通过子提供商和 modelMap 实现模型路由。

入口与路由
- 全局 store，在 get-model composable 和 CustomProvider 视图中使用

核心数据流
1. db.providers.toArray() → useLiveQuery → providers
2. providers → computed providerTypes（合并自定义 + 原生提供商类型）
3. providers → computed modelOptions（合并所有模型名）
4. createProvider(provider, options, stack) → 递归创建 SDK provider 函数

依赖关系
- composables/live-query：响应式查询
- utils/db：Dexie 数据库
- utils/values：ProviderTypes（原生提供商类型列表）、baseModelOptions
- utils/types：CustomProvider, Provider, ProviderType
- @sinclair/typebox：TObject 构造空 Schema

关键实现细节
- L11-26：createProvider 判断 type 是否以 'custom:' 开头来路由到 createCustomProvider 或原生 ProviderTypes
- L15-22：从 localStorage 恢复 apiKey（备份机制）
- L28-40：createCustomProvider 遍历 subproviders 查找 modelMap 匹配，未找到则使用 fallbackProvider，stack 参数防止循环引用
- L55-68：providerTypes computed 将自定义提供商包装为 ProviderType 格式
- L69-72：modelOptions computed 合并所有模型名（包括子提供商的 modelMap 键）

与其他模块的交互
- useGetModel：调用 createProvider 和 getModelList
- CustomProvider 视图：编辑自定义提供商配置
- values.ts ProviderTypes：原生提供商定义
