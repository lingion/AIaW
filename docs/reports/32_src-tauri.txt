模块名称：src-tauri/ — 桌面端壳
文件路径：src-tauri/

功能概述
Tauri 桌面端配置和 Rust 后端。提供原生窗口管理、流式 HTTP 请求（绕过浏览器 CORS 限制）、剪贴板、文件系统、自动更新等能力。支持 Windows/macOS/Linux 三平台。

入口与路由
- src/main.rs：入口，调用 app_lib::run()
- src/lib.rs：Tauri Builder 配置，注册插件和命令
- src/stream.rs：stream_fetch 命令实现
- tauri.conf.json：Tauri 配置（窗口、打包、安全策略）

核心数据流
1. 前端调用 tauriFetch（utils/tauri-stream.ts）→ invoke('stream_fetch') → Rust stream_fetch
2. stream_fetch：reqwest HTTP 请求 → bytes_stream → Tauri emit → 前端 EventTarget 接收
3. 请求 ID（AtomicU32）匹配请求和响应

Rust 代码结构
- lib.rs：
  - 注册插件：window-state, opener, process, clipboard-manager, updater, os, shell
  - invoke_handler：stream::stream_fetch, is_deb::is_deb_package
  - Debug 模式启用 tauri-plugin-log

- stream.rs（源自 NextChat）：
  - L30-32：stream_fetch 接收 method, url, headers, body
  - L48-54：使用 reqwest Client 发送请求
  - L60-76：异步 spawn 读取 bytes_stream，通过 window.emit 发送 ChunkPayload
  - L82-88：流结束时发送 EndPayload
  - L93-113：错误处理同样通过 emit 发送错误信息

Tauri 配置
- productName: "AI as Workspace"
- identifier: "app.aiaw"
- 窗口：1150×750，禁用 dragDrop
- 打包格式：deb, appimage, nsis, app( macOS), dmg
- CSP: null（无限制）

与其他模块的交互
- utils/platform-api.ts：检测 IsTauri，使用 tauriFetch 替代 window.fetch
- utils/tauri-stream.ts：前端 Tauri 流式请求封装
- utils/tauri-shell-transport.ts：MCP STDIO 传输（Tauri 独有）
