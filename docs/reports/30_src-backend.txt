模块名称：src-backend/ — 后端适配层
文件路径：src-backend/

功能概述
基于 FastAPI 的 Python 后端服务。提供 CORS 代理、文档解析、搜索引擎聚合、代码执行沙箱、文件操作等功能。在桌面端（Tauri）作为本地服务运行，在 Web 端部署为独立后端。

入口与路由
- app.py：FastAPI 应用入口
- /cors/proxy：CORS 代理（白名单：search1api, pollinations, web-crawler）
- /doc-parse/parse：LlamaParse 文档解析
- /searxng：SearXNG 搜索引擎代理
- /code-exec：Python 代码执行沙箱
- /file/read, /file/write, /file/list, /file/exec：文件操作
- 最后挂载 static/ 目录（前端 SPA）

核心数据流
1. 前端 fetch → /cors/proxy → aiohttp 转发（白名单校验）
2. 前端上传文件 → /doc-parse/parse → LlamaParse → Markdown
3. 前端 → /searxng → 环境变量 SEARXNG_URL
4. 前端 → /code-exec → tempfile → subprocess Python → stdout/stderr/images
5. 前端 → /file/* → FILE_OPS_ROOT 内文件操作

依赖关系
- requirements.txt：fastapi, aiohttp, llama-parse 等
- aiohttp.ClientSession：HTTP 客户端
- llama_parse：LlamaParse API

关键实现细节
- L19-21：ALLOWED_PREFIXES 白名单限制代理目标
- L77-93：LlamaParse 异步解析文档
- L99-110：SearXNG 代理转发查询参数
- L118-162：代码执行沙箱：自动注入 matplotlib 图表保存 hook，base64 编码输出图片
- L170-173：FILE_OPS_ROOT 限制文件操作范围，_resolve() 防止路径穿越
- L229：最后挂载 StaticFiles 兜底 SPA 路由

与其他模块的交互
- utils/cors-fetch.ts：前端调用 /cors/proxy
- utils/doc-parse-plugin.ts：前端调用 /doc-parse/parse
- utils/code-exec-plugin.ts：前端调用 /code-exec
- utils/file-ops-plugin.ts：前端调用 /file/*
- utils/web-search-plugin.ts：前端调用 /searxng
