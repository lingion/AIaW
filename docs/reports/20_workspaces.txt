模块名称：stores/workspaces.ts（工作区状态管理）
文件路径：src/stores/workspaces.ts

功能概述
工作区（Workspace）和文件夹（Folder）的 Pinia store。使用 Dexie LiveQuery 实现响应式数据库查询。提供工作区/文件夹的 CRUD 操作，删除工作区时级联删除所有关联数据（对话、消息、条目、助手、Artifact）。

入口与路由
- 全局 store，在视图和 composables 中按需导入
- useWorkspacesStore()

核心数据流
1. db.workspaces.toArray() → useLiveQuery → workspaces（响应式数组）
2. addWorkspace/addFolder → db.workspaces.add
3. putItem → db.workspaces.put（全量替换）
4. updateItem → db.workspaces.update（部分更新）
5. deleteItem → 递归级联删除

依赖关系
- composables/live-query：useLiveQuery 包装 Dexie liveQuery
- utils/db：Dexie 数据库实例
- utils/types：Workspace, Folder 类型
- utils/templates：DefaultWsIndexContent
- utils/functions：genId

关键实现细节
- L7：workspaces 使用 useLiveQuery 自动响应 IndexedDB 变更
- L10-23：addWorkspace 创建完整默认值的工作区，包含 id、name、avatar、type、parentId、indexContent、vars、listOpen
- L35-46：deleteItem 递归删除子项，然后在一个事务中级联删除 dialogs、messages、items、assistants、artifacts

与其他模块的交互
- WorkspaceIndex/WorkspaceSettings：读取和修改工作区数据
- AssistantsStore：工作区关联助手
- DialogView：工作区关联对话
- useWorkspaceActions：提供创建/重命名/删除等 UI 操作
